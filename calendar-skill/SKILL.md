# 日历/日记系统（记忆系统中层）

这是 Claude Code 三层记忆模型的**中期存储层**。日历数据保存在 `~/.claude/calendar/`。

```
三层记忆模型：
  短期（会话上下文）→ 中期（日历日志）→ 长期（MEMORY.md）
                                              ↑
                              每 2-3 天空闲时自动提炼
```

## 目录结构

```
~/.claude/calendar/
└── YYYY-MM-DD/
    ├── 摘要.md          # 当日任务列表（一行一条）
    └── 完整记录/         # 逐任务完整对话
        ├── 01.md
        ├── 02.md
        └── auto-HHMMSS.md  # 异常退出自动保存
```

## 读取日历

### 查看最近记录

列出 `~/.claude/calendar/` 下最近 7 天的目录，显示每个日期的摘要。用 `Glob` 找到所有 `*/摘要.md` 文件，按日期倒序排列。

### 查看某一天

用户指定日期后：
1. 先读取 `${date}/摘要.md` 展示任务列表
2. 用户对某个任务感兴趣时，读取 `完整记录/XX.md` 展示完整内容

### 搜索（四级回退）

当用户想查找之前的某项工作时，按以下优先级搜索：

**级别 1：搜索摘要文件（最快）**
```bash
grep -rl "关键词" ~/.claude/calendar/*/摘要.md
```

**级别 2：展示命中的摘要条目**

找到匹配的摘要文件后，读取该文件，只展示匹配关键词的条目行。如果只有一个结果，直接展示该条目；如果有多个结果，列出所有匹配让用户选择。

**级别 3：搜索完整记录**
```bash
grep -rl "关键词" ~/.claude/calendar/*/完整记录/*.md
```
命中后直接展示相关片段（grep 加 `-C 2` 显示上下文）。

**级别 4：搜索长期记忆**
```bash
grep -rl "关键词" ~/.claude/projects/-home-$(whoami)/memory/*.md
```

**注意事项：**
- 先用摘要搜索（快、命中率高），没结果再逐级回退
- 如果只命中一条摘要，直接展示该条目内容
- 展示搜索结果时标注日期和任务编号
- 前三级在日历中搜，第四级在长期记忆文件中搜

## 保存到日历

### 创建今日摘要

由用户口述当天做了什么，或由你根据当前会话内容总结。创建 `~/.claude/calendar/YYYY-MM-DD/` 目录和 `摘要.md` 文件。

**摘要.md 格式：**
```markdown
# YYYY-MM-DD 工作记录

1. 简短描述任务一（见完整记录 01.md）
2. 简短描述任务二（见完整记录 02.md）
```

每句话控制在 1-2 行，只记录关键信息。

### 保存完整记录

将用户指定的任务对话内容保存到 `完整记录/NN.md`。索引编号与摘要中的编号一致。

如果用户没有指定任务内容，只创建摘要即可。

## 记忆提炼（长短期桥接）

这是日历系统与长期记忆系统的桥接逻辑。

### 触发条件

满足以下任一条件时执行：
- 距离上次提炼超过 3 天
- 用户说"提炼记忆"、"更新长期记忆"、"总结最近"
- 当日历累计了 5 天以上的新日志

### 提炼流程

1. 读取最近 3-7 天的 `摘要.md` 文件
2. 识别值得长期保存的信息，按类型归类：

| 信息类型 | 目标文件 |
|----------|----------|
| 用户新偏好、习惯、工具选择 | `user-prefs.md` |
| 项目决策、架构选择、进展更新 | `project-context.md` |
| 踩坑经历、经验教训 | `feedback-lessons.md` |
| 新发现的外部资源、路径、入口 | `reference-index.md` |
| 用户新技能、角色变化 | `user-profile.md` |

3. 更新对应的记忆文件（只追加新信息，删除已过时的信息）
4. 更新 `MEMORY.md` 索引

### 提炼格式

每个条目遵循以下格式：
```markdown
## YYYY-MM-DD：[主题]

**教训/事实：** [一句话概括]

**Why:** [为什么重要]

**How to apply:** [未来遇到类似情况该怎么做]
```

## 自动保存

会话结束时系统自动保存转录到日历。自动保存的文件名格式为 `auto-HHMMSS.md`。

## 注意事项

- 日期使用北京时区 (Asia/Shanghai)
- 摘要文件用中文书写
- 手动保存的完整记录索引从 01 开始递增
- 自动保存的文件以 `auto-` 前缀命名
- **只追加不覆盖** — 这是核心规则
- **不凭记忆猜测** — 回答历史问题前必须先搜索
- 不要创建空目录或空文件
