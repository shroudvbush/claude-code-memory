# CLAUDE.md — Memory System Rules

将以下内容追加到你已有的 `~/.claude/CLAUDE.md` 文件中。

## 5. Memory System (持续性记忆)

Claude Code 自带记忆系统。以下规则定义了何时以及如何使用它。

### 记忆三层模型

| 层级 | 位置 | 内容 | 生命周期 |
|------|------|------|----------|
| **短期** | 会话上下文 | 当前对话 | 会话内 |
| **中期** | `~/.claude/calendar/YYYY-MM-DD/` | 每日会话原始记录 | 永久追加 |
| **长期** | `~/.claude/projects/-home-$USER/memory/` | 提炼后的结构化记忆 | 持续精炼 |

### 会话启动（每次必做）

1. 读取 `MEMORY.md` 索引 — 了解有哪些长期记忆
2. 读取 `user-profile.md` — 知道在和谁合作
3. 读取今天的 `~/.claude/calendar/YYYY-MM-DD/摘要.md`（如果存在）— 知道今天做了什么

### 会话进行中（写日志）

每当完成一项重要工作后，追加到今天的日历日志：
- 文件：`~/.claude/calendar/YYYY-MM-DD/摘要.md`
- 格式：编号 + 一句话描述（见 calendar skill 的详细规范）
- 规则：只追加不覆盖，不创建空文件

### 会话结束时

总结当日完成的工作，确保日历摘要已更新。

### 记忆维护（每 2-3 天一次，空闲时）

1. 读取最近 3 天的 `~/.claude/calendar/*/摘要.md`
2. 识别值得长期保存的信息，按类型归类：

| 信息类型 | 目标文件 |
|----------|----------|
| 用户新偏好、习惯、工具选择 | `user-prefs.md` |
| 项目决策、架构选择、进展更新 | `project-context.md` |
| 踩坑经历、经验教训 | `feedback-lessons.md` |
| 新发现的外部资源、路径、入口 | `reference-index.md` |
| 用户新技能、角色变化 | `user-profile.md` |

3. 更新对应的记忆文件（只追加新信息，删除已过时的信息）
4. 更新 `MEMORY.md` 索引

### 回答历史问题之前

当用户问"之前怎么做的"、"有没有遇到过"、"上次那个"等问题时：
1. 先搜日历摘要 — `grep -rl "关键词" ~/.claude/calendar/*/摘要.md`
2. 再搜完整记录 — `grep -rl "关键词" ~/.claude/calendar/*/完整记录/*.md`
3. 最后查长期记忆 — 如果日历无结果，搜索 `memory/*.md`
4. 基于证据回答，不凭记忆猜测

### 记忆安全红线

- 不导出私有数据
- 危险操作前必须确认
- 不确定时主动问，不猜测
